Installation of TSpaces-Server:
------------------------------

TSpaces can be retrieved for free from http://www.almaden.ibm.com/cs/tspaces/

installation: 
- unpack the file "tspaces212.zip"
- add the folders "tspaces\classes" and the file "tspaces\lib\tspaces.jar" to the CLASSPATH
- the server can be started with the command: "java  com.ibm.tspaces.server.TSServer [options]"
Note: the server demands a configuration file "tspaces.cfg" that is expected to be in the location where the server is started. It can be specified with the option [-c ConfigurationFile]

Additional notes for the V2-Installation:

1)	Install the "tspaces.jar" into the "maven repository":
	Use following command within the windows console within the directory with the tspaces.jar:
	
	“mvn install:install-file -Dfile=tspaces.jar -DgroupId=com.tspaces -DartifactId=tspaces2 -
	Dversion=2.1.2 -Dpackaging=jar –e” --> eventuell auch ohne "-e" versuchen.

2)	Add following piece of code to the pom.file of the jadex.launch project:
			<dependency>
			<groupId>com.tspaces</groupId>
			<artifactId>tspaces2</artifactId>
			<version>2.1.2</version>
			<scope>compile</scope>
			</dependency>

3) Add the „tspaces.cfg“ file to the jadex.launch directory.

4) The TSpaces-Server has to know the classes, i.e. TupleContent.class, that are stored within the
   Tuples. Therefore:
	1) Export the dcm-project as "jar" using eclipse: export-> as jar --> whole project
	2) "mvn install:install-file -Dfile=decomas.jar -DgroupId=org.decomas -DartifactId=decomas -Dversion=1.2 -Dpackaging=jar"
	3) Add following piece of code to the pom.file of the jadex.launch project: 
			<dependency>
			<groupId>org.decomas</groupId>
			<artifactId>decomas</artifactId>
			<version>1.2</version>
			<scope>compile</scope>
			</dependency>
